# 会议室逃脱游戏说明文档

场景路径：

`Assets/Scenes/ConferenceRoom_before_blockout_sync.unity`

## 游戏简介

这是一个第一人称会议室逃脱解谜场景。玩家被锁在会议室内，需要探索房间中的关键物体，收集线索到背包中，推理出四位门禁密码，在门旁键盘输入密码，打开门并离开会议室。

当前版本保留了已有会议室场景布局，没有重建整个房间。玩家需要围绕会议桌、椅子、柜子、电脑、便签纸和门禁键盘进行探索。

## 操作方式

| 按键 | 功能 |
| --- | --- |
| 鼠标 | 控制视角 |
| WASD | 玩家移动 |
| E | 与视野中心的物体互动 |
| Tab | 打开或关闭背包 |
| 背包打开时按 E | 切换到下一条线索 |
| 键盘输入模式下 0-9 | 输入门禁数字 |
| 键盘输入模式下 Backspace | 删除上一位数字 |
| 键盘输入模式下 Enter | 提交密码 |
| 键盘输入模式下 Escape | 取消输入 |

## 基本玩法流程

1. 玩家进入会议室并探索环境。
2. 看向关键物体并按 `E` 互动。
3. 有些物体会把线索加入背包。
4. 按 `Tab` 打开背包查看已收集线索。
5. 根据线索推理四位门禁密码。
6. 与 `keypad_01` 互动，进入密码输入模式。
7. 输入正确密码后，`locked_door_01` 会打开。
8. 穿过 `victory_exit_point` 后游戏胜利。

## 背包系统

背包用于存放玩家收集到的线索。

- 玩家视角上方会显示一个 `Bag` 图标。
- 按 `Tab` 打开或关闭背包。
- 已收集线索显示在背包左侧。
- 当前选中的线索内容显示在右侧纸条区域。
- 背包打开时按 `E` 可以切换到下一条线索。
- 也可以用鼠标点击左侧线索条目进行切换。
- 背包支持富文本颜色，例如红色重点提示。

当玩家获得新线索时，会显示提示：

`Clue Found! Note added to backpack.`

该提示会显示一小段时间，然后逐渐淡出。

当前可加入背包的线索：

| 物体 | 背包条目 | 作用 |
| --- | --- | --- |
| `cabinet_01` | `Cabinet Note` | 提供密码第一位相关信息。 |
| `seat_card_01` 到 `seat_card_04` | `Seat Order Note` | 记录会议座位顺序线索。 |
| `desktop_computer_01` | `Computer Hint` | 显示 `Hint: the password is ?O??`，其中 `?O??` 为红色提示。 |

## 倒计时与胜负条件

游戏总时间限制为 10 分钟，也就是 600 秒。

- Play Mode 开始后计时开始。
- 玩家视角边缘会显示倒计时进度条。
- 进度条满格代表 600 秒。
- 时间经过时，进度条会逐渐缩短。
- 文本以 `MM:SS` 格式显示剩余时间，例如 `Time Left: 09:42`。
- 剩余时间较少时，倒计时显示会变得更醒目。

胜利条件：

- 玩家必须先解锁并打开 `locked_door_01`。
- 然后穿过 `victory_exit_point`。
- 胜利时显示：

`Game Success! You escaped the meeting room.`

失败条件：

- 如果 600 秒内没有逃出房间，游戏失败。
- 失败时显示：

`Game Failed! Time limit exceeded.`

## 物体互动系统

玩家互动使用第一人称视角射线检测。

- 看向可互动物体。
- 按 `E`。
- Unity Console 会打印该物体的 `object_id` 和描述。
- 如果该物体带有线索组件，线索会加入背包。

重要物体 ID：

| Object ID | 作用 |
| --- | --- |
| `locked_door_01` | 锁住的出口门 |
| `keypad_01` | 密码输入键盘 |
| `cabinet_01` | 可拉开的柜子，也是线索来源 |
| `desktop_computer_01` | 电脑线索来源 |
| `seat_card_01` 到 `seat_card_04` | 座位顺序线索 |
| `sticky_note_01` | 便签纸引导目标 |
| `victory_exit_point` | 开门后触发胜利的出口区域 |

## 定时引导系统

如果玩家长时间没有与关键物体互动，系统会逐步给出视觉引导。引导包含物体高亮和从玩家位置连到目标物体的线。

引导只会在玩家尚未与目标物体按 `E` 互动过时出现。

| 时间 | 引导目标 |
| --- | --- |
| 60 秒 | 高亮并连线 `seat_card_01` 到 `seat_card_04` |
| 120 秒 | 高亮并连线 `desktop_computer_01` |
| 180 秒 | 高亮并连线 `cabinet_01` |
| 240 秒 | 高亮并连线 `sticky_note_01` |

当玩家与被引导目标互动后，对应引导会隐藏。

## 密码键盘与门

玩家与 `keypad_01` 互动后，会进入密码输入模式。

输入模式中：

- 按数字键输入密码。
- 按 Backspace 删除上一位。
- 按 Enter 提交。
- 按 Escape 取消输入。

如果密码错误，会打印错误提示并清空输入，玩家可以重新尝试。

如果密码正确，`locked_door_01` 会打开，玩家可以走向出口。

## 建议测试流程

1. 进入 Play Mode。
2. 确认倒计时进度条出现。
3. 按 `Tab`，确认背包可以打开和关闭。
4. 与柜子互动，确认线索加入背包。
5. 与电脑互动，确认电脑线索加入背包。
6. 打开背包，按 `E` 确认可以切换线索。
7. 等待定时引导触发，确认未互动目标会高亮并连线。
8. 与被引导目标互动，确认对应引导隐藏。
9. 与键盘互动并输入正确密码。
10. 门打开后穿过 `victory_exit_point`，确认胜利提示出现。
11. 如需测试失败流程，可以临时缩短 `GameStateManager.timeLimitSeconds`。
